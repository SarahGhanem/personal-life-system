# Feature docs

One markdown file per feature, written before that feature is implemented (see the Feature Workflow section in the root `CLAUDE.md`). Name files by feature slug, e.g. `docs/features/expense-categories.md`.

Copy `TEMPLATE.md` to start a new one.

This folder started after several features already existed in the app (auth, calendar events, weekly goals, daily notes, weekly reflection, monthly finance, monthly reflection) — those aren't retroactively documented here. Everything built from this point forward gets a doc.
